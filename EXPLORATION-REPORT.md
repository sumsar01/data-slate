# Data-Slate Codebase Exploration Report

## Executive Summary

Data-Slate is a **Warhammer 40K themed campaign logging system** for recording, organizing, and analyzing tabletop RPG session notes. The system uses:
- **Frontend:** React 19 + Vite + TypeScript
- **Backend:** Hono (web framework) on Bun + TypeScript
- **Database:** Turso (libsql/SQLite)
- **Storage:** Cloudflare R2 (S3-compatible)
- **AI:** Groq LLM SDK

# 1. ADMIN PANEL UI (apps/web)

## Main Admin Component
**File:** `/Users/rasmus/git/github.com/sumsar01/data-slate/apps/web/src/pages/Admin.tsx`

### Key Features
The admin panel (`/admin-mechanicus` route) provides:

1. **DATA EXPORT**
   - Export all sessions as markdown
   - Export individual sessions by date

2. **SHARE LINKS**
   - Create shareable tokens for sessions
   - Manage active share links
   - Revoke access

3. **ENTITY INDEX STATUS**
   - Count total notes
   - Track which notes have extracted entities
   - Retry entity extraction for missing cases

4. **TRANSCRIPT FLAVOURING**
   - Batch rewrite all transcripts with Warhammer 40K terminology
   - Re-extract entities after processing

5. **ENTITY WIKI**
   - Sync entities from all transcripts
   - Edit entity type (NPC/Location/Faction/Item/Other)
   - Add descriptions
   - Upload images to R2
   - Generate Groq "dossier" summaries
   - Merge duplicate entities

### Admin Component Structure
```typescript
interface WikiEntity {

  image_url: string | null      // R2 URL
```

### Admin State Management
```typescript
- groups: DateGroup[]                  // All sessions
```

```typescript

  id: string                    // UUID
  date: string                  // YYYY-MM-DD

  duration_s: number            // Recording length in seconds
  tags: Tag[]                   // User-selected tags
  entities: Entity[]            // Extracted via Groq NER
  created_at: string            // ISO timestamp

```

### Tag Types (9 available)
```typescript
type Tag = 
  | "NPC"
  | "Location"
  | "Faction"
  | "Combat"
  | "Clue"
  | "Tech-Lore"
  | "Rumour"
  | "Item"

const ALL_TAGS: Tag[] = [
  "NPC", "Location", "Faction", "Combat", "Clue",
  "Tech-Lore", "Rumour", "Item", "Misc"
]
```

### Entity Type (Extracted from transcripts)
```typescript
type Entity = {
  name: string                  // Entity name
  type: EntityType              // NPC | Location | Faction | Item | Other
}

type EntityType = "NPC" | "Location" | "Faction" | "Item" | "Other"
```

### DateGroup (Notes grouped by date)
```typescript
type DateGroup = {
  date: string                  // YYYY-MM-DD
  session_id: string | null     // FK to session_overrides
  session_name: string | null   // User-provided name
  session_summary: string | null // Groq-generated summary
  notes: Note[]                 // All notes for this date
}
```

### SessionOverride (Campaign/Session metadata)
```typescript
type SessionOverride = {
  id: string                    // UUID
  name: string                  // User-provided session name
  dates: string[]               // Array of YYYY-MM-DD dates in session
  summary?: string | null       // Groq-generated battle report
}
```

# 3. API ENDPOINTS FOR LOG ENTRIES

**Base URL:** `${API_URL}` (typically `http://localhost:3001` or deployed Railway instance)

## Notes Router
**File:** `/Users/rasmus/git/github.com/sumsar01/data-slate/apps/api/src/routes/notes.ts`

### POST /notes — Upload & Create Note
**Request:**
```
Content-Type: multipart/form-data

Fields:
- audio: File (audio/webm or audio/mp4)
- date: string (YYYY-MM-DD)
- duration_s: number (seconds)
- tags: string[] (Tag names)
```

**Response:** `201 Created`
```typescript
{
  id: string
  date: string
  title: string                 // Auto-generated
  transcript: string            // Flavoured
  audio_url: string | null
  duration_s: number
  tags: Tag[]
  created_at: string
}
```

**Pipeline:**
1. Upload audio to R2
2. Transcribe via Groq (detect language)
3. Flavour transcript with 40K terminology (keeping original language)
4. Generate English title from flavoured transcript
5. Fire-and-forget: Extract entities via Groq NER

---

### GET /notes — List All Notes
**Response:** `200 OK`
```typescript
Note[]
```
Sorted by `created_at DESC`

---

### GET /notes/:id — Get Single Note
**Response:** `200 OK`
```typescript
Note
```
Returns 404 if not found.

---

### POST /notes/:id/entities — Retry Entity Extraction
**Request:** `Content-Type: application/json`
```typescript
{
  transcript: string
}
```

**Response:** `200 OK`
```typescript
{
  id: string
  entities: Entity[]
}
```

Used by admin to manually re-extract entities for a note.

---

### POST /notes/flavour-all — Batch Flavour All Notes
**Request:** No body

**Response:** `200 OK`
```typescript
{
  total: number
  processed: number
  failed: number
}
```

**Process:**
- Iterates all notes in creation order
- Detects language from text (self-referential)
- Regenerates title from flavoured text
- Re-extracts entities

---

### DELETE /notes/:id — Delete Note
**Response:** `200 OK`
```typescript

}
```

Also attempts to delete audio from R2 (best-effort, warns if fails).

# 5. DATABASE SCHEMA

**System:** Turso (SQLite with HTTP API)

## Tables

### notes
```sql
CREATE TABLE notes (
  id TEXT PRIMARY KEY,
  date TEXT NOT NULL,                 -- YYYY-MM-DD
  title TEXT NOT NULL,                -- Generated by Groq
  transcript TEXT NOT NULL,           -- Flavoured
  audio_url TEXT,                     -- R2 public URL
  duration_s REAL NOT NULL,
  tags TEXT NOT NULL,                 -- JSON: ["NPC", "Faction"]
  entities TEXT,                      -- JSON: [{ name, type }]
  created_at TEXT NOT NULL            -- ISO timestamp
)
```

### session_overrides
```sql
CREATE TABLE session_overrides (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  dates TEXT NOT NULL,                -- JSON: ["2024-01-15", "2024-01-16"]
  summary TEXT                        -- Groq-generated
)
```

### entities (wiki)
```sql
CREATE TABLE entities (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL UNIQUE,
  type TEXT NOT NULL,                 -- NPC | Location | Faction | Item | Other
  canonical_id TEXT,                  -- FK for merged duplicates
  description TEXT,
  summary TEXT,                       -- Groq dossier
  image_url TEXT,                     -- R2 URL
  created_at TEXT NOT NULL
)
```

### shares
```sql
CREATE TABLE shares (
  id TEXT PRIMARY KEY,
  token TEXT NOT NULL UNIQUE,         -- 32-char UUID without dashes
  session_id TEXT NOT NULL,           -- FK to session_overrides.id
  created_at TEXT NOT NULL,
  expires_at TEXT,                    -- ISO timestamp (nullable)
  FOREIGN KEY (session_id) REFERENCES session_overrides(id)
)
```

# 7. EDIT/CREATE PATTERNS

## Current Editing Capabilities

### What CAN be edited:
1. **Session name** — via SessionOverride component (inline)
2. **Entity metadata** — via Admin panel:
   - Type (NPC/Location/Faction/Item/Other)
   - Description (text input)
   - Image (upload/remove)
   - Merge duplicates

3. **Notes cannot be edited directly** — only deleted
   - Once created, transcript/title/tags are immutable
   - Must delete and re-record if correction needed

### What CANNOT be edited:
- Note transcript content
- Note title (auto-generated, would need re-transcription)
- Note tags (fixed at creation)
- Note date
- Note audio URL
- Note duration
