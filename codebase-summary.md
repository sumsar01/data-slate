# Data-Slate Codebase Structure Summary

## Overview
Data-Slate is a 40K-themed campaign management system with audio recording, transcription, and entity tracking. Built with React 19 + Vite (frontend), Hono (backend), Turso SQLite (database), and Groq LLM (AI).

---

## 1. Campaign/Session Structure

### Current Terminology
The system uses **"sessions"** (not campaigns) to group notes by date/narrative arc. These are called **SessionOverride** because they override/name date groupings.

### Key Types (Frontend)
**File:** `/apps/web/src/shared.ts`

```typescript
type DateGroup = {
  date: string                 // YYYY-MM-DD
  session_id: string | null    // UUID of session_overrides row
  session_name: string | null  // User-friendly name (e.g., "Session 1: The Void Born")
  session_summary: string | null // Groq-generated summary of session notes
  notes: Note[]                // All notes for that date
}

type SessionOverride = {
  id: string                   // UUID
  dates: string[]              // Array of YYYY-MM-DD strings
  name: string                 // Display name
}

type Note = {
  id: string
  date: string                 // YYYY-MM-DD
  title: string                // Generated by Groq
  transcript: string           // Flavoured with 40K terminology
  audio_url: string | null     // Cloudflare R2 URL
  duration_s: number
  tags: Tag[]                  // NPC, Location, Faction, Combat, Clue, Tech-Lore, Rumour, Item, Misc
  entities: Entity[]           // Extracted by Groq NER
  created_at: string           // ISO timestamp
}

type Entity = {
  name: string
  type: EntityType             // NPC | Location | Faction | Item | Other
}
```

## 2. Database Schema (Turso SQLite)

### Tables

#### `notes`
```sql
CREATE TABLE notes (
  id TEXT PRIMARY KEY,
  date TEXT NOT NULL,              -- YYYY-MM-DD
  title TEXT NOT NULL,              -- Generated by Groq
  transcript TEXT NOT NULL,        -- Flavoured
  audio_url TEXT,                  -- Cloudflare R2 public URL
  duration_s REAL NOT NULL,
  tags TEXT NOT NULL,              -- JSON array of Tag strings
  entities TEXT,                   -- JSON array of Entity objects
  created_at TEXT NOT NULL         -- ISO timestamp
)
```

#### `session_overrides`
```sql
CREATE TABLE session_overrides (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,              -- Display name
  dates TEXT NOT NULL,             -- JSON array ["YYYY-MM-DD", ...]
  summary TEXT                     -- Groq-generated summary
)
```

#### `shares`
```sql
CREATE TABLE shares (
  id TEXT PRIMARY KEY,
  token TEXT NOT NULL,             -- UUID without dashes (32 chars)
  session_id TEXT NOT NULL,        -- FK to session_overrides.id
  created_at TEXT NOT NULL,        -- ISO timestamp
  expires_at TEXT                  -- ISO timestamp (nullable)
)
```

#### `entities` (Wiki)
```sql
CREATE TABLE entities (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,              -- Entity name
  type TEXT NOT NULL,              -- NPC | Location | Faction | Item | Other
  canonical_id TEXT,               -- FK to entities.id (for deduplication)
  description TEXT,                -- User-editable
  summary TEXT,                    -- Groq dossier
  image_url TEXT,                  -- Cloudflare R2 URL
  created_at TEXT NOT NULL         -- ISO timestamp
)
```

## 3. API Routes & Endpoints

**Base:** `http://serene-analysis-production-145c.up.railway.app` (production)

### Health
- `GET /health` → `{ status: "ok", ts: timestamp }`

### Notes
- `POST /notes` — Upload audio, transcribe, store
  - Body: `FormData { audio: File, date: string, duration_s: number, tags: Tag[] }`
  - Response: Full Note object
- `GET /notes` — List all notes
- `GET /notes/:id` — Get single note
- `POST /notes/:id/entities` — Retry entity extraction
- `POST /notes/flavour-all` — Retroactively flavour all transcripts & re-extract entities
  - Response: `{ total, processed, failed }`
- `DELETE /notes/:id` — Delete note & cleanup R2 audio

### Dates (Aggregate)
- `GET /dates` — List all DateGroup[] with sessions/summaries merged
  - Returns: `DateGroup[]` with notes grouped by date

### Sessions
- `POST /sessions` — Create session override
  - Body: `{ name: string, dates: string[] }`
  - Response: Session object
- `PATCH /sessions/:id` — Update session name/dates
- `POST /sessions/:id/summary` — Generate & store session summary via Groq
  - Response: `{ id, summary }`

### Shares
- `POST /shares` — Create read-only share link
  - Body: `{ session_id: string, expires_in_days?: number }`
  - Response: `{ id, token, session_id, created_at, expires_at }`
- `GET /shares` — List all shares (admin only)
- `DELETE /shares/:id` — Revoke share
- `GET /shares/shared/:token` — Public access to shared session
  - Returns: `{ session_name, groups: DateGroup[] }`

### Wiki (Entity Index)
- `GET /wiki` — List all canonical entities
- `GET /wiki/by-name/:name` — Lookup entity by name
- `GET /wiki/:id` — Entity detail with note excerpts
- `POST /wiki/sync` — Scan all transcripts, populate entity index
  - Response: `{ inserted }`
- `POST /wiki/:id/summary` — Generate Groq dossier
- `PATCH /wiki/:id` — Update entity (type, description, etc.)
- `POST /wiki/:id/image` — Upload entity image
- `DELETE /wiki/:id/image` — Remove entity image
- `POST /wiki/merge` — Merge duplicate entities
  - Body: `{ keep_id, drop_id }`

## 4. Frontend Routing & Page Structure

**File:** `/apps/web/src/main.tsx`

```typescript
Routes:
```

### Pages

#### `/` — Main Log (App.tsx)
**File:** `/apps/web/src/App.tsx`
- Left panel: NoteList (grouped by date, with session naming)
- Right panel: NoteReader (selected note detail)
- Features:
  - Tag filtering
  - Full-text search
  - Session override UI (SessionOverride component)
  - Entity index sidebar
  - Export to markdown

#### `/record` — Audio Recorder (Record.tsx)
**File:** `/apps/web/src/pages/Record.tsx`
- WebRTC audio capture
- Real-time duration tracking
- Tag selection
- Upload to API
- Redirect to main view on success

#### `/admin-mechanicus` — Admin Panel (Admin.tsx)
**File:** `/apps/web/src/pages/Admin.tsx`
- **Export section:** Download sessions as markdown
- **Share links section:** Create/manage read-only share tokens
- **Entity extraction status:** Track missing entities, retry failed extractions
- **Transcript flavouring:** Batch reprocess transcripts with 40K terminology
- **Entity Wiki:** 
  - Sync entities from transcripts
  - Edit entity types, descriptions
  - Upload entity images
  - Generate Groq dossiers
  - Merge duplicate entities

#### `/share/:token` — Public Share (ShareView.tsx)
**File:** `/apps/web/src/pages/ShareView.tsx`
- Read-only view of shared session
- Shows session name & grouped notes
- No editing capabilities

#### `/wiki` — Entity Browser (Wiki.tsx)
**File:** `/apps/web/src/pages/Wiki.tsx`
- List all entities (NPCs, Locations, Factions, Items)
- Search/filter
- Click to view entity detail

#### `/wiki/:id` — Entity Detail (WikiPage.tsx)
**File:** `/apps/web/src/pages/WikiPage.tsx`
- Entity bio + description
- Groq-generated dossier
- Note excerpts mentioning entity
- Related entities

## 6. Data Flow & API Integration

### Frontend → Backend
**File:** `/apps/web/src/data/api.ts`

```typescript
// Session management
upsertSession(date, name, existingId?) 
  → POST /sessions or PATCH /sessions/:id

generateSummary(sessionId)
  → POST /sessions/:id/summary

// Note management
deleteNote(id)
  → DELETE /notes/:id
```

### Data Fetching
**File:** `/apps/web/src/hooks/useDateGroups.ts`
- Polls `/dates` endpoint every 30 seconds
- Falls back to mock data if API unavailable
- Used throughout app via `useDateGroups()` hook

## 9. Terminology Notes

### "Campaign" vs "Session"
In code, the system uses:
- **Session** / **SessionOverride** - A named grouping of one or more dates
- A session can span multiple dates (e.g., "Campaign Arc: The Void Born" = dates 2024-01-15 through 2024-01-22)
- Each date has 0+ notes (audio recordings)

### 40K Flavor
- Notes are called "records" in UI ("AUSPEX LOG", "COGITATOR RECORD")
- Transcripts are "flavoured" with Warhammer 40K terminology
- Entities are stored in a "WIKI" / "DOSSIER" system
- Share links are called "read-only shares" for distribution
